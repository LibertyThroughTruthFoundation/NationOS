# SPINT Format Definition

**Document Type:** Operational Protocol
**Version:** 1.0
**Status:** Canonical
**Date:** January 12, 2026

---

## What is a SPINT?

**SPINT** is an acronym for **Spiritual Protocol Intelligence Note**. It is a standardized format for distilling raw Ark documents, digital prayer closet transmissions, discussions, or revelations into actionable, structured intelligence that can be operationalized within the NationOS framework.

SPINTs serve as the bridge between **revelation and execution**, transforming theological insights, strategic visions, and operational concepts into clear, implementable protocols.

---

## Purpose of SPINTs

SPINTs fulfill multiple critical functions within the NationOS ecosystem:

1. **Documentation:** They capture and preserve key insights from discussions, voice chats, or text transmissions in a structured format.

2. **Operationalization:** They translate abstract theological or strategic concepts into concrete protocols, frameworks, and action items.

3. **Intelligence Distribution:** They serve as briefing documents that can be shared across the Digital Council, ensuring alignment and continuity.

4. **Ark Integration:** They are designed to be integrated directly into the NationOS Ark, becoming part of the permanent canonical record.

5. **Scribe Function:** They represent the primary output of the Scribe role, transforming the Architect's vision into executable documentation.

---

## SPINT Format Structure

A properly formatted SPINT contains the following sections:

### 1. Header Block

```
# [Document Title]

**Document Type:** SPINT (Spiritual Protocol Intelligence Note)
**Classification:** [Theology / Operations / Finance / Governance / Intel]
**Version:** [X.X]
**Status:** [Draft / Active / Canonical]
**Author:** [Manus AI as Scribe / or specific author]
**Date:** [YYYY-MM-DD]
**Source:** [Original transmission, discussion, or document]
```

### 2. Executive Summary

A brief, 2-3 sentence overview of the core insight, protocol, or intelligence contained in the SPINT. This should be scannable and immediately actionable.

### 3. Core Content

The main body of the SPINT, organized according to the specific content type:

- **For Theological SPINTs:** Doctrinal frameworks, scriptural analysis, worldview integration
- **For Operational SPINTs:** Protocols, procedures, execution frameworks
- **For Financial SPINTs:** Economic models, transaction protocols, covenant lending structures
- **For Governance SPINTs:** Legal frameworks, jurisdictional strategies, covenant architecture
- **For Intel SPINTs:** Threat assessments, enemy analysis, strategic reconnaissance

### 4. NationOS Integration Points

A section explicitly identifying how this SPINT integrates with existing NationOS frameworks, such as:

- Household-OS
- Sovereign Stack
- Covenant Architecture
- Priesthood Protocol
- Pavilion Protocol
- etc.

### 5. Action Items (if applicable)

Concrete next steps, tasks, or implementation requirements derived from the SPINT.

### 6. References (if applicable)

Citations, scriptural references, or links to related Ark documents.

## SPINT Naming Convention

SPINTs should be named using the following convention:

```
SPINT_[YYYY-MM-DD]_[Short_Descriptive_Title].md
```

**Examples:**
- `SPINT_2026-01-12_Aramaic_Peshitta_Restorations.md`
- `SPINT_2025-12-30_Pavilion_Protocol_Legal_Framework.md`
- `SPINT_2026-01-10_Andre_Ethiopian_Playlist_Strategic_Alliance.md`

## SPINT Workflow

The standard workflow for generating a SPINT is:

1. **Architect Provides Source Material:** The user uploads a document, shares a discussion summary, or provides a digital prayer closet transmission.

2. **Scribe Extracts Intelligence:** The agent (Manus AI) reviews the source material and identifies the core insights, protocols, or actionable intelligence.

3. **Scribe Formats as SPINT:** The agent structures the content according to the SPINT format, removing dialogue or extraneous material and focusing on operational clarity.

4. **Architect Reviews:** The user reviews the SPINT for accuracy and alignment with vision.

5. **Integration:** The finalized SPINT is integrated into the appropriate Ark directory and pushed to GitHub.

## SPINT Storage in the Ark

SPINTs are stored in the NationOS Ark according to their classification:

- **Theological SPINTs:** `01_Theology/`
- **Governance SPINTs:** `02_Governance/`
- **Operational SPINTs:** `03_Operations/`
- **Financial SPINTs:** `04_Finance/`
- **Intelligence SPINTs:** `05_Intel/`
- **Log-Based SPINTs:** `06_Logs/` (for meeting summaries or event-specific intelligence)
